# AI & Data Science Job Market (2020–2026)

An interactive data visualization project on the AI and Data Science job market,
built as a Streamlit dashboard backed by scripted, reproducible figures. It is
based on the ["AI & Data Science Job Market Dataset (2020–2026)"](https://www.kaggle.com/datasets/shree0910/ai-and-data-science-job-market-dataset-20202026),
a **synthetic** dataset generated with statistical distributions and controlled
randomness. The data is therefore illustrative: it is not a source on the real
labour market, and patterns should be read as properties of the simulation.

Authors: Mohamed Amine El Hani, Angelica Fiorio, Francesco Leoni.

## Key findings

- The market is **flat**: both the volume of openings and the mix of roles are
  stable across all seven years. There is no growth and no role overtaking another.
- Pay is driven by a few clear factors: **seniority**, **job title** (engineering
  roles pay a premium), **company size** (only multinationals stand out), and the
  **advanced skills** Machine Learning, Deep Learning and Cloud.
- Several expected factors turned out to carry **no signal**: country, industry,
  remote type, education (a PhD pays no more than a Bachelor), and the two most
  requested skills, Python and SQL.

## Project structure

```
project/
  app/
    streamlit_app.py     the interactive dashboard (4 tabs + global filters)
  data/
    processed/           jobs_clean.parquet, produced by preprocessing.py
    raw/                 original CSV (immutable input)
  report/
    figures/             the 8 PNG figures generated by plots.py (fig01–fig08)
  src/
    plots.py             the 8 static figures used in the technical report
    preprocessing.py     quality checks, cleaning, derived columns, quality assertions
  pyproject.toml         project metadata and dependencies
  README.md
  uv.lock                pinned dependency versions (reproducible install)
```

## Requirements

- Python ≥ 3.11
- [uv](https://docs.astral.sh/uv/) for dependency management and running the code

All library versions are pinned in `uv.lock`, so `uv sync` reproduces the exact
environment (pandas, matplotlib, seaborn, plotly, streamlit, and pyarrow for the
parquet file).

## Setup and usage

```bash
uv sync                                    # install the pinned dependencies
uv run python src/preprocessing.py         # run quality checks and build the clean table
uv run python src/plots.py                 # regenerate the 8 report figures
uv run streamlit run app/streamlit_app.py  # launch the interactive dashboard
```

Run `preprocessing.py` first: it reads the raw CSV, validates it, and writes
`data/processed/jobs_clean.parquet`, which both `plots.py` and the dashboard read.

## The dashboard

The Streamlit app has a sidebar with global filters (posting year, countries, job
titles) and a permanent note about the synthetic data, plus four tabs:

1. **Market evolution**: how the role mix changes over time (it stays flat).
2. **Skills & salary**: the core view: salary by skill and seniority, and the
   salary premium of each skill.
3. **Global landscape**: a bubble map with a companion table, showing the
   geographic uniformity of the market.
4. **Success formula**: what actually predicts pay (seniority and company size),
   with education shown as a null result.

Every aggregate is weighted by `job_openings`, since each posting advertises 1–9
positions. A colourblind-safe palette with one fixed colour per role is used
consistently across the figures and the dashboard.
